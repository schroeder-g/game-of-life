Welcome to Cube of Life! This manual explains how to interact with the simulation. The application has two main modes: View Mode for observing, and Edit Mode for modifying the grid.

Main Header Controls

Mode Toggle (✏️/📽️): Switches between Edit Mode (Pencil icon), where you can modify the grid, and View Mode (Projector icon), where you can run the simulation and freely move the camera.

References: src/contexts/BrushContext.tsx, src/contexts/SimulationContext.tsx

Test IDs: UC-1

Playback Controls (▶/⏸, ⏮/⏭, ↺): Play/Pause the simulation (Spacebar), step forward/backward one generation (→/←), or reset the grid to its initial state (R). Step and Reset are only available when paused.

References: src/core/Grid3D.ts, src/contexts/SimulationContext.tsx

Test IDs: UC-2

Camera Actions (Fit/Recenter): Use 'Fit' (F) to zoom and frame all live cells. 'Recenter' (S) moves the camera to the origin.

References: src/contexts/SimulationContext.tsx

Test IDs: UC-11

Select Scene (🖼️): Opens a dropdown to quickly load pre-made or user-saved scenes, including their patterns and rules.

Test IDs: UC-3

Documentation (?): Opens this user manual. (Shortcut: ? or Shift+/)

References: src/components/DocumentationModal.tsx

Test IDs: UI-2

Shortcuts Help: Opens a modal displaying all keyboard shortcuts for both View and Edit modes. The overlay can be closed by clicking its backdrop or pressing 'Escape'.

References: src/components/ShortcutOverlay.tsx

Test IDs: UC-12

Camera Controls & Rotation

Camera Rotation: Use the rotation keys (O/Period for Pitch, K/Semicolon for Yaw, I/P for Roll) for smooth, continuous 'flight-sim' style rotation. This is the only rotation mode, optimized for intuitive navigation.

Test IDs: UC-11

Special Rules in Edit Mode: When a brush shape is active (e.g., 'Cube', 'Sphere'), the rotation keys will rotate the brush instead of the camera. To rotate the camera while a brush is active, you must use the Ctrl/Shift override.

Test IDs: UX-1

Ctrl/Shift Override: At any time, holding Ctrl or Shift (or both) while pressing a rotation key will temporarily perform smooth, continuous camera rotation, even if a brush is active.

Test IDs: UX-5

Editing the Grid (Edit Mode)

Paint/Clear Modes (+/-): Select Paint (+) to add cells or Clear (-) to remove them. You can click-and-drag on the grid, or use the Spacebar (paint) and Delete (clear) keys to apply the current brush.

References: src/contexts/BrushContext.tsx, src/contexts/SimulationContext.tsx

Test IDs: UC-4

Select Brush (🖌️): Opens a dropdown to choose a brush shape, such as Cube or Sphere. You can also select a 'Community' of cells from the grid to use as a custom brush.

References: src/contexts/BrushContext.tsx, src/core/Grid3D.ts

Test IDs: UC-5

Configuration Panel

Speed: In View Mode, this slider controls the simulation speed in generations per second.

References: src/contexts/SimulationContext.tsx

Test IDs: UC-6

Environment Section: Controls the grid's properties. Adjust 'Grid Size' and 'Cell Margin'. 'Reset' reverts to the initial state, 'Clear' removes all cells, and 'Random' populates the grid based on the 'Density' slider.

References: src/contexts/SimulationContext.tsx, src/core/Grid3D.ts

Test IDs: UC-7

Rules Section: Defines the automaton's logic. Set the neighbor count for a cell to 'Survive' or for a new cell to be 'Born'. Checkboxes for Faces, Edges, and Corners determine which neighbor types are counted.

References: src/contexts/SimulationContext.tsx

Test IDs: UC-8

Cursor Position Section: Provides inputs to set the editor cursor's X, Y, and Z coordinates. It also shows the keyboard shortcuts for moving the cursor, which change based on your camera angle.

References: src/contexts/BrushContext.tsx, src/contexts/SimulationContext.tsx

Test IDs: UC-9

Shape Brush Section: Customize the selected brush's 'Size' and 'Hollow' properties. Brush rotation is controlled with the 'I' and 'P' keys. When using destructive tools like the 'Clear Brush', the rotation direction for 'I' and 'P' is reversed for intuitive control.

References: src/core/shapes.ts, src/contexts/BrushContext.tsx, src/hooks/useAppShortcuts.ts

Test IDs: UX-1

Scene Management Section: Save the current grid, rules, and settings as a new named scene. You can also 'Export' scenes to a file, 'Import' them from a file, or 'Delete' saved scenes.

Test IDs: UC-10

Camera Controls Section: Adjust sensitivity for Pan, Yaw (swivel), and Roll. You can invert controls and set camera easing (acceleration/deceleration) for smoother movements.

Test IDs: UC-11

Architectural & Quality Claims

To prevent accidental actions while typing, global keyboard shortcuts are disabled when an input field (e.g., in the Configuration Panel or the Welcome Modal) is focused. This ensures typing does not trigger camera movement or simulation controls.

References: src/hooks/useAppShortcuts.ts, src/components/WelcomeModal.tsx

Test IDs: QA-1, UX-7

The file `core/faceOrientationKeyMapping.ts` is critical for predictable camera and brush orientation. It must not be modified without a deep understanding of its spatial rotation logic. This ensures that user controls for navigation remain consistent and reliable.

References: src/core/faceOrientationKeyMapping.ts

Test IDs: QA-2

This project adheres to a strict quality process. Every feature or claim documented in this manual is cross-referenced with one or more manual tests (e.g., UX-1, QA-1). This ensures all documented functionality is verifiable and that the documentation stays in sync with the application's behavior. (`src/data/documentation.ts`, `src/data/manual-tests.ts`)

Test IDs: QA-3

Data Structure Integrity (DS-1): The core Grid3D data structure shall reliably manage cell states. It must initialize correctly, handle out-of-bounds access gracefully, accurately set, get, and toggle cell states, and support clearing, saving, and restoring its state without data loss. Its simulation tick must correctly apply the rules of life.

References: src/core/Grid3D.ts

Brush Controls Panel Draggability

The BrushControls panel (src/components/BrushControls.tsx) is designed to be freely draggable anywhere on the screen. Users can click and drag the panel to reposition it according to their preference. This functionality is implemented using mouse and touch event listeners (handleMouseDown, handleMouseMove, handleMouseUp, handleTouchStart, handleTouchMove, handleTouchEnd) that update the panel's position state.

Test IDs: TEST_BC_DRAG_001

Brush Controls Panel Initialization

Upon initialization, the BrushControls panel (src/components/BrushControls.tsx) automatically centers itself on the screen. This ensures a consistent and user-friendly starting layout, regardless of screen size or previous usage. The initial positioning logic is handled within a useEffect hook that calculates the center coordinates based on window.innerWidth and window.innerHeight.

Test IDs: TEST_BC_INIT_001

Brush Controls Panel Appearance

The BrushControls panel features a clear "Brush Controls" header, a subtle orange outline, and rounded corners for a modern and integrated look. The header text is styled with color: #FFA500 and the panel border with border: 2px solid #FFA50080 and borderRadius: '8px' in src/components/BrushControls.tsx.

Test IDs: TEST_BC_APPEAR_001

Brush Controls Panel Width

The BrushControls panel is designed to be 50% wider than its default size, providing more space for its controls. This is achieved by setting the maxWidth property of its internal grid container to '300px' in src/components/BrushControls.tsx.

Test IDs: TEST_BC_APPEAR_002

Brush Controls Panel Visibility

The BrushControls panel is only visible when the simulation is in edit mode (i.e., when viewMode is false). This conditional rendering is managed in src/components/AppHeaderPanel.tsx, ensuring the controls do not clutter the interface during camera rotation.

Test IDs: TEST_BC_VIS_001

Brush Controls Directional Buttons

The directional buttons (Up, Down, Left, Right, Further, Closer) on the BrushControls panel (src/components/BrushControls.tsx) allow precise manipulation of the brush selector. When clicked, these buttons emit a moveSelector event via the eventBus with a delta value determined by the current cameraOrientation, ensuring movement is relative to the user's view.

Test IDs: TEST_BC_BUTTON_001_W, TEST_BC_BUTTON_001_X, TEST_BC_BUTTON_001_A, TEST_BC_BUTTON_001_D, TEST_BC_BUTTON_001_Q, TEST_BC_BUTTON_001_Z

Application Header Panel

The AppHeaderPanel (src/components/AppHeaderPanel.tsx) serves as the main control and information hub at the top of the screen. It is divided into three main sections: the title, the status panel, and the control buttons area.

References: src/components/AppHeaderPanel.tsx

Title and Build Information

The left section of the header displays the application title, "Cube of Life". Below the title, it shows build information, including the version number and build time. In non-production builds, it also displays a personalized welcome message if a username is set.

References: src/components/AppHeaderPanel.tsx

Test IDs: AHP_TITLE_001

Live Status Panel

The central status panel provides real-time information about the simulation state. It displays the current scene name, the camera's face orientation and rotation, the active brush shape (in Edit Mode), and live simulation statistics like generation count and number of living cells.

References: src/components/AppHeaderPanel.tsx

Test IDs: AHP_STATUS_001

Contextual Control Visibility

Controls are displayed contextually. The Play/Pause button and Speed slider (UC-6) are only available in View mode. 'Step' or 'Reset' buttons are disabled while the simulation runs. The community icon appears only when a community is selected in Edit mode.

The Controls.tsx component is deprecated and no longer in use. Its functionality, which previously included the community selection sidebar, has been moved to SelectedCommunityPanel.tsx. This documentation entry is retained for historical purposes.

Manual Tests Panel

Available in non-production builds, the ManualTestsPanel (src/components/ManualTestsPanel.tsx) allows developers and testers to track the verification status of features. Users can cycle a test's status between 'unchecked', 'checked', and 'failed'. Test details, including steps and associated documentation claims, can be viewed by expanding each test item. Statuses are persisted in local storage under the key 'manual-tests-statuses', as managed by the useManualTests hook.

References: src/components/ManualTestsPanel.tsx, src/hooks/useManualTests.ts
